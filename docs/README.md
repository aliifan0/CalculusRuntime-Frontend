# Frontend Documentation

This folder documents the `frontend/` subproject only.

## Purpose

The frontend is the CalcVoyager React single-page app. It provides:

- Study guide pages for simple concepts, partial derivatives, and vector calculus.
- Interactive tools for continuity, extrema, and volume/integral calculations.
- Login, signup, dashboard, bookmarks, quiz state, and progress tracking UI.
- A hosted AI solver launch page.

## Stack

- React
- Create React App / `react-scripts`
- React Router
- KaTeX for math rendering
- Math-related packages such as `mathjs`, `crosstex`, `tex-math-parser`, and `slangmath`

## Runtime Configuration

The frontend reads `REACT_APP_API_URL`.

Current default in code:

```env
REACT_APP_API_URL=http://127.0.0.1:8002
```

Set this explicitly for deployments so production builds do not accidentally target localhost.

## Math Rendering Notes

The app uses two math rendering paths:

- Direct components: `InlineMath` and `BlockMath` from `src/components/Math.jsx`.
- Guide auto-rendering: `StudyGuideShell.jsx` renders `$...$`, `$$...$$`, `\(...\)`, and `\[...\]` inside guide content.

When adding guide content:

- Keep each LaTeX expression inside one complete delimiter pair.
- Prefer `$x < 0$` as one text string instead of splitting `$x `, `<`, and ` 0$` across multiple JSX nodes.
- Use valid LaTeX commands such as `\to`, `\neq`, `\leq`, and `\geq`.
- Run `npm run build` after edits.

## Current Technical Debt

- Progress data is still mostly local to `localStorage`; backend sync should be added for logged-in users.
- Some route names are prototype-style, such as `/test`, `/extreme`, and `/volumecalculator`.
- `stats.totalSections` in `ProgressContext` is hardcoded and should be derived from shared curriculum metadata.
- Test coverage is thin for auth, dashboard, progress, guide quizzes, and tool behavior.
